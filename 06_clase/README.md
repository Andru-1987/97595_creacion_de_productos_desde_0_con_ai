## Resumen del curso: Feedback, métricas y crecimiento para startups

### 1. ¿Qué es una startup?
Una startup no es una empresa pequeña. Es una organización temporal que busca un **modelo de negocio escalable y repetible**. Su objetivo no es solo construir un producto, sino aprender rápido si ese producto puede crecer. La mentalidad de crecimiento se basa en experimentar, validar hipótesis y pivotar o perseverar con evidencia.

### 2. Hipótesis de negocio y producto
Una hipótesis es una afirmación comprobable. Estructura básica:
- Creencia: "Creemos que [usuario] tiene [problema]".
- Experimento: "Ofreceremos [solución mínima]".
- Métrica de éxito: "Tendremos razón si [métrica] alcanza X".

### 3. Feedback: cualitativo vs cuantitativo
- **Cualitativo** (entrevistas, grabaciones de sesión): explica el "por qué". Ideal para detectar problemas y fricciones.
- **Cuantitativo** (analytics, métricas): muestra el "qué". Ideal para medir tendencias y comportamientos masivos.
- Regla práctica: combínalos (triangulación). Usa datos cuantitativos para encontrar caídas y cualitativos para entenderlas.

### 4. Early adopters y Design Partners
- **Early adopters**: usuarios que sufren el problema, toleran imperfecciones y dan feedback profundo. Son el primer mercado objetivo.
- **Design Partners**: colaboran activamente en el diseño y validación a cambio de acceso anticipado o condiciones especiales. Ayudan a co-crear soluciones reales.
- Ambos se reclutan mejor con contacto directo y personalizado (WhatsApp, Slack). Sé claro, empático y respeta la privacidad.

### 5. North Star Metric (NSM)
Es la métrica que mejor captura el valor central que entregas. No es una métrica de vanidad.
- Ejemplos: noches reservadas (Airbnb), mensajes enviados (WhatsApp), tiempo de consumo (Netflix).
- Debe ser: medible, adelantada (predecir éxito futuro) y accionable (el equipo puede influir en ella).

### 6. Producto Mínimo Viable (MVP)
El MVP no es una versión barata, sino el experimento más pequeño para aprender. Ejemplos:
- Airbnb: fotos de calidad de sus propios departamentos para validar confianza.
- Buffer: una landing page con un botón de precio para validar intención de pago sin desarrollar el producto.

### 7. Herramientas de session recording (Hotjar, FullStory)
Permiten grabar sesiones reales, generar mapas de calor y analizar embudos. Sirven para detectar dónde los usuarios se pierden o abandonan. Configúralas sin código (copiar snippet en el sitio) y enfócate en eventos clave (clics en botones, envíos de formularios).

### 8. Dashboards y KPIs prácticos
Un buen dashboard evita métricas de vanidad. Enfócate en:
- Activación: ¿los usuarios tienen el momento "ajá" pronto?
- Retención: ¿la curva se aplana o cae a cero?
- Engagement: ¿qué usuarios son frecuentes?
Usa gráficos simples (línea para tendencias, embudo para caídas, tarjeta de número grande para la NSM). Vincula el dashboard con las grabaciones de sesión para investigar caídas.

### 9. Señales de Product-Market Fit (PMF)
- Cuantitativas: crecimiento sostenido de usuarios activos, retención alta, conversión en acciones clave.
- Cualitativas: feedback positivo recurrente, recomendaciones espontáneas, usuarios que exploran funciones avanzadas.

### 10. Plan de iteración hacia PMF
Ciclo continuo:
1. Define objetivos basados en hipótesis.
2. Recolecta datos (cuantitativos y cualitativos con Design Partners y herramientas de sesión).
3. Analiza patrones y valida o invalida hipótesis.
4. Decide si pivotas o mejoras.
5. Implementa cambios y mide el impacto.
